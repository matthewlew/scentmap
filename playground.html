Magic Numbers & Off-Grid Values

These break the typography scale, the 4px spacing grid, or the semantic token contract. Fix before any future PR touching these lines.

VIOLATION — app.js:1137 
<div class="card-meta"
  style="font-size:10px; margin-bottom:var(--sp-xs);">
  + ${p.b.name}
</div>
SOLUTION 
<!-- card-meta already has fs-meta in CSS; -->
<!-- remove inline style entirely -->
<div class="card-meta">
  + ${p.b.name}
</div>

10px is below the type scale. .card-meta in CSS already sets var(--fs-meta). Strip the inline style — no CSS change needed.

VIOLATION — app.js:1409, 3159, 3918 
<span style="color:var(--accent);
  margin-left:2px;
  font-size:0.85em;
  text-decoration:none;
  display:inline-block;">★</span>
SOLUTION — new .saved-mark class 
/* Add to components.css */
.saved-mark {
  color: var(--accent-primary);
  margin-left: var(--sp-xs);
  font-size: var(--fs-meta);
  text-decoration: none;
  display: inline-block;
  flex-shrink: 0;
}

<!-- In JS -->
<span class="saved-mark">★</span>

Used in 3 places. 2px margin violates grid; 0.85em is relative sizing. Consolidate to one class.

VIOLATION — app.js:2065, 2192 
<div style="font-size:var(--fs-meta);

var(--g500) is a primitive grey. Use var(--text-tertiary) (semantic). .panel-eyebrow already has correct type and spacing.

.s-name-btn (components.css:817) and .link-btn are functionally identical: both are text-only ghost buttons with underline and hover transition. They have different visual weight settings (s-name-btn is fs-body/500, link-btn is inherited/500) but the same intent. Consolidate to .link-btn and remove the duplicate.

CURRENT — duplicate definitions 
/* components.css:817 — one-off */
.s-name-btn {

  font-weight: 500;
  color: var(--text-primary);

  text-underline-offset: var(--link-underline-offset);
  text-decoration-color: var(--g300);
  transition: ...;
}

/* components.css — canonical .link-btn */
.link-btn {
  /* same intent, different specifics */
}
MIGRATION 
/* 1. Audit all .s-name-btn usages in app.js.
      Replace with .link-btn.
   2. Delete .s-name-btn from components.css.
