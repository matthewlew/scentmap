Scentmap System

Source of truth for all components, styles, and patterns used across the application. Any design changes should be updated here and verified.

System Audit & Proposals

1-off Hardcoded Pixel Values

The following hardcoded pixels were found in the CSS and should be consolidated or converted to variables where possible. Note: There are currently 0 hardcoded hex colors!

  • components.css: ~80 hardcoded pixels.
    Mostly small adjustments (1px, 2px, 3px) for optical alignment or fixed geometry items like dots (8px) and structural dividers (1px, 2px, 4px).
  • layout.css: ~15 hardcoded pixels.
    Mainly structural constraints like max-width (500px), header clearance (80px), or specific tap-targets (44px) and structural radii (16px, 2px).
  • responsive.css: ~16 hardcoded pixels.
    Mostly media queries, base font sizes (17px, 18px), and geometry updates for mobile like tap target heights (44px, 62px) and dot sizes (8px, 9px).
  • design-system.css: ~2 hardcoded pixels.
    Focus ring width (2px) and base HTML font-size (16px).

Proposals for Consolidation

1. Create standard variables for fine-tuning

Values like 1px, 2px, and 3px are used heavily for optical alignments. We should add `--sp-micro: 2px` or `--sp-1px: 1px` to the spacing scale to formalize this.

2. Unify "Dot" components

There are multiple declarations for 8px/10px/9px dots (e.g., `.cmp-dd-dot`, `.fam-dot`, `.picker-fdot`). Consolidate these into a single `.ui-dot` component with standard sizes.

3. Standardize fixed sizes

A few specific numbers keep appearing (44px for touch targets, 16px for radii). We should use standard tokens like `--touch-target-min: 44px` and `--radius-xl: 16px`.
