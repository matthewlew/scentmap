Design System Consistency

How do all 4 components use the same typography, sizing, and corner radius?
Every component in components.css references tokens from design-system.css only — never hardcoded values. The token scales are the enforcement mechanism. If you're writing a new component, the rule is simple: if the value doesn't have a token, it doesn't belong in the component.

Typography
Font family → var(--font-sans)
Font size → var(--fs-caption)
var(--fs-meta)
var(--fs-body)
Weight → 600 (UI element standard)
or 500 (secondary)
❌ No raw 12px, 14px, etc.
Corner Radius
Pill → var(--radius-pill) (20px)
Tag → var(--radius-sm) (5px)
Tab → calc(var(--radius) - 2px)
(inset from container)
❌ No raw border-radius: 4px
Spacing
Padding → var(--sp-xs) through var(--sp-md)
Gap → var(--sp-2xs) or var(--sp-sm)
Heights → component-specific (24px chip, 44px tab) — documented as exceptions
❌ No raw padding: 4px 8px

The current one-offs break this rule. Classes like .saved-brand-chip, .picker-order-badge, and .notes-nav-btn define their own redundant typography and spacing instead of inheriting from the canonical component. Migration fixes this by collapsing them to components that already use the token system correctly.
